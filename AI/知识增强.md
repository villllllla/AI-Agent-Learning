### 1. 检索增强生成 (Retrieval-Augmented Generation, RAG)
采用非参数化记忆（Non-parametric Memory）扩展机制，这是目前解决知识幻觉（Hallucination）和时效性问题的最优工程实践。
- **核心机制：** * **离线阶段（Data Ingestion）：** 将增量语料集执行 Chunking（分块），利用 Embedding 模型（如 BGE, text-embedding-ada-002）将其映射为高维稠密向量（Dense Vectors），并构建基于近似最近邻算法（如 HNSW, FAISS 索引）的向量数据库（Vector Database）。
    - **在线推理阶段（Inference）：** 将 User Query 进行同样的 Embedding 编码，通过余弦相似度（Cosine Similarity）等度量方式检索 Top-K 相关的语料片段。
    - **上下文注入（Context Injection）：** 将检索到的上下文与原始 Prompt 拼接，依赖 LLM 的上下文学习（In-Context Learning, ICL）能力，利用 Transformer 的 Self-Attention 机制对引入的新实体和逻辑进行即时计算与解码。
        
### 2. 上下文窗口扩展 (Context Window Scaling)
不改变模型权重，通过提升底层架构对长序列信息的吞吐上限，实现海量增量数据的直接输入（Dynamic Contextual Injection）。
- **核心机制：** 突破传统 Transformer 架构中 Attention 机制 $O(N^2)$ 的时间/空间复杂度瓶颈，以及位置编码（Position Encoding）的外推限制。
    - **位置编码优化：** 采用旋转位置编码（RoPE）的动态缩放技术，如 NTK-aware Scaling 或 YaRN，实现对未见过的长序列位置信息的泛化。
    - **显存与计算优化：** 结合 FlashAttention 等底层 IO 感知算法，以及 PagedAttention 对 KV Cache（键值缓存）的分页内存管理，将 Context Length 推升至 100k 甚至 1M+ Tokens。使模型能够在单次 Inference 生命周期内，处理极大规模的文档级别新知识。

### 3. 参数高效微调 (Parameter-Efficient Fine-Tuning, PEFT)
更新模型的**参数化记忆**，将特定领域或时效性知识编码进神经网络权重中。
- **核心机制：** 全参微调（Full-Parameter Tuning）算力成本极高且极易引发**灾难性遗忘（Catastrophic Forgetting）**。当前主要采用以 LoRA (Low-Rank Adaptation) 为代表的低秩自适应技术。
- **算法原理：** 在冻结预训练基础模型权重矩阵 $W \in \mathbb{R}^{d \times k}$ 的前提下，旁路注入两个低秩可训练矩阵 $A \in \mathbb{R}^{r \times k}$ 和 $B \in \mathbb{R}^{d \times r}$ （其中秩 $r \ll \min(d, k)$）。前向传播时的计算变为 $h = Wx + \Delta Wx = Wx + BAx$。
- **应用：** 训练时反向传播仅更新矩阵 $A$ 和 $B$ 的参数。此方法常结合监督微调（Supervised Fine-Tuning, SFT），以低计算代价实现模型在特定垂直领域的知识注入与行为对齐（Alignment）。
    
### 4. 持续预训练 (Continual Pre-Training, CPT)
针对模型底层表示空间（Representation Space）与全局世界模型（World Model）的深度重构。
- **核心机制：** 对冻结的 Base Model 重新开启自回归（Autoregressive）预训练过程，使其在极大尺度的增量 Token 集合上继续拟合下一个词的概率分布。
- **工程难点（Continual Learning）：** 为缓解参数剧烈更新导致的灾难性遗忘效应，必须精心设计数据混合策略（Data Mixture），引入**历史重放缓冲区（Replay Buffer）**，将一定比例的原始 Pre-training 数据与增量数据混合。同时需重新初始化学习率调度器（如引入 Learning Rate Rewarm-up 机制），使梯度在复杂损失地形（Loss Landscape）中跳出局部最优，平滑吸收新的基础知识。
---

**技术选型矩阵 (Trade-off Matrix):**

| **增强机制**            | **知识存储介质**             | **算力与工程成本** | **遗忘风险** | **主要解决场景**             |
| ------------------- | ---------------------- | ----------- | -------- | ---------------------- |
| **RAG**             | 向量数据库 (Non-parametric) | 极低          | 无        | 强事实性问答、高频实时数据更新        |
| **Context Scaling** | KV Cache (Transient)   | 推理时显存消耗高    | 无        | 极长文档的零样本分析 (Zero-shot) |
| **PEFT (LoRA)**     | 低秩权重矩阵 (Parametric)    | 中等 (需训练)    | 低        | 垂直领域术语学习、特定范式对齐        |
| **CPT**             | 全局模型权重 (Parametric)    | 极高 (需大规模集群) | 极高       | 语言能力扩充、底层常识库全面升级       |
