### 一、 发生在哪里？（硬件与架构层面）

很多人以为对比计算发生在大模型里，其实完全不是。大模型只负责“把文字变成向量”，而**向量对比计算，100% 发生在你部署的“向量数据库（Vector DB）”内部。**
具体到物理硬件上，它发生在这里：
#### 1. 必须在 RAM（运行内存）中进行
这是向量数据库极其吃内存的根本原因。浮点数计算要求极高的吞吐量，如果每次对比都要去 SSD 硬盘里读数据，速度会慢几千倍。所以，像 Milvus、Qdrant 这样的数据库，在启动时会**把海量的向量和索引图全部加载到内存（RAM）中**。
#### 2. 在 CPU 的 SIMD 指令集 或 GPU 中计算
- **CPU 计算（最常见）：** 现代 CPU 有一个专做这种苦力活的加速机制，叫 **SIMD**（单指令流多数据流，比如 Intel 的 AVX-512 指令集）。它可以让你在一条 CPU 指令内，同时对多个浮点数进行乘法和加法运算，极大地榨干 CPU 的并行计算能力。
- **[GPU 计算](AI-Agent-Learning/AI/CUDA并行计算.md)（土豪玩法）：** 如果你的向量库规模极大（上亿级别），且要求极高的 QPS（每秒查询率），像 Milvus 就支持把向量加载到显存里，利用 Nvidia GPU 极其恐怖的几千个 CUDA 核心进行矩阵并发计算。


### 二、 如何进行计算？（数学层面）
#### 1. 余弦相似度 (Cosine Similarity) —— NLP 领域的绝对王者
大模型的文本向量，**99% 的情况下都在使用余弦相似度**。
- **原理：** 它不看两个点在空间里离得多远，**只看这两个向量的“夹角”**。
- **为什么适合文本：** 一篇 1000 字讲“苹果手机”的文章，和一句 10 个字讲“苹果手机”的短语，把它们变成向量后，长度（大小）可能差很多，但它们的**方向（语义）是一致的**。余弦相似度就是用来消除文本长度带来的干扰，只比对“方向”。
- **公式：** $Cosine(A, B) = \frac{A \cdot B}{||A|| \times ||B||}$
- **结果判定：** 夹角越小，值越接近 `1`（极其相似）；夹角 90 度，值为 `0`（毫不相干）；夹角 180 度，值为 `-1`（意思完全相反）。
- **适用场景**：绝大多数的大语言模型（LLM）文本特征提取，如 OpenAI 的 `text-embedding` 系列、开源的 BGE 模型。
- **选择原因**：文本生成的向量，其“方向”代表了语义，而“长度”往往受到句子长短或词汇出现频率的干扰。余弦相似度自带归一化效果，完全消除了长度的干扰。
#### 2. 内积 (Dot Product
- **特点：** 它是余弦相似度的“简配版”。如果你提前把所有向量的长度都归一化（Normalize）成 1（即**L2 归一化 (L2 Normalization)**），那么内积的计算结果和余弦相似度完全一样，但**内积的计算速度要快得多**，因为它省去了除法和开根号的过程。
- **适用场景**：**这是工业界生产环境的终极首选！** 但前提是：**必须先在代码里把所有向量进行 L2 归一化 (L2 Normalization)**。
- **数学原理**：如果向量 $A$ 和 $B$ 的长度都已经缩放为 1（即 $||A||_2 = 1, ||B||_2 = 1$），那么它们的余弦相似度计算公式中，分母直接变成了 1。此时：**Cosine = Dot Product**。
	- 为什么要缩放为1：
	1. **内积（Dot Product）是硬件的最爱**：现代 CPU 和 GPU 内部都有专门的 **FMA（乘加融合指令）**，它们一个时钟周期就能同时做完“乘法和加法”。内积纯粹就是乘加组合，硬件跑起来就像高铁一样平顺。
	2. **除法和开平方是“性能刺客”**：在底层芯片中，除法和开平方运算需要消耗几十个甚至上百个时钟周期，会严重阻断 CPU 的流水线并行。
	    ```Python
	    import numpy as np
		# 假设这是大模型吐出来的、未经归一化的 1536 维向量
		v = np.array([0.015, -0.024, 0.089, ...]) 
		# 核心代码：向量自己 / 自己的 L2 范数
		v_normalized = v / np.linalg.norm(v)
		# 存入 Qdrant / Milvus / Pinecone 数据库...
	    ```
	所以，如果你的数据库里有 1 亿个向量，用带开根号的余弦公式去算，哪怕用 HNSW 算法，服务器可能也会因为 CPU 负载过高而发烫。但如果全换成内积，性能可以直接**翻好几倍**。
- **选择原因**：算余弦需要做极其耗时的平方根和除法运算，而算内积只需要纯粹的乘法和加法。归一化后的内积计算，能够完美利用 GPU 的 Tensor Cores 或 CPU 的 SIMD 指令集，**性能相比原始余弦计算可以提升数倍**。
#### 3. 欧氏距离 (Euclidean Distance / L2)
- **原理：** 也就是我们在初中数学里学的“两点之间的直线距离”。
- **特点：** 它既看方向，也看大小。通常在图像检索（CV）或者推荐系统里用得比较多，但在纯文本的大模型语义检索里用得相对较少。
